The Hidden Risk

The Real Enemy Is Context Drift

The longer a project runs, the more the AI loses the plot.

  • Specs go stale — the AI's map no longer matches the territory
  • The AI pattern-matches locally instead of navigating purposefully
  • Developers and agents diverge silently, accumulating invisible debt

Context drift is when the AI's working model of the project no longer matches reality — leading to increasingly off-target output that compounds over time.

Common Mistake

The "One Big AGENTS.md" Mistake

Most teams start with one giant instruction file. It breaks in predictable ways:

  • Context is scarce — a huge file crowds out the actual task
  • When everything is "important," nothing is
  • It rots instantly — stale rules, silent drift
  • Hard to verify coverage, freshness, or ownership

Treat AGENTS.md as a table of contents, not an encyclopedia. Each file owns one concern. The agent knows where to look.

The Fix

Structured Context, Not a Monolith

. ├── AGENTS.md ← navigation only, not an encyclopedia ├── ARCHITECTURE.md ← system decisions, ADRs └── docs/ ├── specs/ │ └── YYYY-MM-DD-<topic>-design.md ← one spec per feature └── generated/ └── db-schema.md ← auto-generated, always fresh
