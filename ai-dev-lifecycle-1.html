The Problem

AI code is inconsistent.
Every session starts from zero.

  • Chat-based development produces code that drifts session to session
  • AI lacks persistent context about architecture and naming conventions
  • Without structure, every prompt is a lottery

"Vague prompts mean wrong results — context quality determines output quality at scale."

— Core insight
03 / 15
The Solution

A three-phase lifecycle
that gives AI the context it needs

01
Bootstrap

Create a comprehensive README that becomes the source of truth for all future AI sessions.

02
Execute

PRD-driven autonomous loop — AI implements feature specs, runs quality checks, commits, and repeats.

03
Ship

Every merge to main is a potential release. CI/CD automates the rest.

Context engineering, not prompt engineering.

Phase Two

Execute

Write a PRD for each feature as a .spec/<feature>.md file. The AI picks stories, implements them, runs checks, commits, and loops until done.

Branch Name
Reference for which git branch this spec targets
Overview
One-paragraph summary of what this feature does
User Stories
As a [user], I want [action] so that [outcome] — with acceptance criteria
Non-Functional Reqs
Latency targets, throughput, error rates, security constraints
Out of Scope
Explicit exclusions to prevent scope creep

The Bigger Picture
Prompt Engineering

Context Engineering
is the new skill.

Developers are shifting from writing code to managing autonomous agents. The leverage isn't in how you phrase the prompt — it's in the quality of the context you give the agent to work within. README, PRD, progress files — these are the new developer artifacts.

13 / 15
Eyes Open

Constraints are real.
They're addressable.

  • Tokens Long runs hit context limits — structure your PRDs to keep individual story implementations within token budget
  • Security Running with elevated permissions (--dangerously-skip-permissions) requires Docker containment for safety
  • Done? Definition of done must be explicit — ambiguous acceptance criteria produce ambiguous implementations
  • Drift AI can still diverge from intent — the review step is the human safeguard that keeps things on track
14 / 15
The Mechanics

Why consistency emerges

Shared ground truth

README + PRD give every session the same starting point. No accumulated drift between feature cycles.

Automated quality gates

Lint, typecheck, and tests run every iteration. The loop only commits when all checks pass.

Resumable by design

Progress files mean interruptions are cheap. The loop picks up exactly where it stopped.

Human leverage

You define what's built and review what was built. The AI handles all the implementation in between.
